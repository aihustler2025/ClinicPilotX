# ClinicPilotX User Guide Draft

Purpose: capture product explanations and step-by-step tutorial notes while the app is being audited and tested.

This is not final customer-facing documentation yet. It is the working source for future:

- Admin user guide.
- Staff user guide.
- Module-by-module tutorials.
- Help center articles.
- Marketing website help/support pages.
- Future screenshot walkthroughs.
- Future video tutorial scripts.

## Documentation Principles

- Write for clinic owners, managers, and front desk staff, not developers.
- Explain what the user sees, what it means, and what action to take next.
- Capture screenshots later after the UI is stable.
- Keep setup/admin documentation separate from daily staff workflows.
- Mark anything unverified until tested in the live app.

## Tutorial Format

Each module tutorial should eventually include:

1. What this module is for.
2. Who uses it.
3. Important terms.
4. Step-by-step daily workflow.
5. Admin/setup workflow.
6. Common mistakes and safety notes.
7. Screenshots or short clips.
8. Related automations.
